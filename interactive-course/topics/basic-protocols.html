What are Web Protocols?

Web protocols are standardized rules and conventions that govern how data is transmitted between web browsers and servers. They ensure reliable, secure, and efficient communication across the internet.

The Web Protocol Stack

Application Layer

HTTP HTTPS FTP SMTP

Transport Layer

Network Layer

Physical Layer

Ethernet WiFi

HTTP - HyperText Transfer Protocol

HTTP is the foundation of data communication on the World Wide Web. It's a stateless, request-response protocol used for transferring web pages and resources.

Stateless

Each request is independent; server doesn't retain client information

Request-Response

Client sends request, server sends response

Text-Based

Human-readable message format

Connectionless

Connection closes after each request-response cycle

HTTP Request Structure

GET /index.html HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8
Accept-Language: en-US,en;q=0.5
Accept-Encoding: gzip, deflate
Connection: keep-alive

[Request Body - for POST/PUT requests]

HTTP Response Structure

HTTP/1.1 200 OK
Date: Mon, 11 Sep 2025 10:30:00 GMT
Server: Apache/2.4.41
Content-Type: text/html; charset=UTF-8
Content-Length: 1234
Last-Modified: Mon, 11 Sep 2025 09:00:00 GMT
ETag: "4f80f-13c-3e1cb03b"

<!DOCTYPE html>
<html>
<head>
    <title>Example Page</title>
</head>
<body>
    <h1>Hello World!</h1>
</body>
</html>

HTTP Methods

HTTP methods indicate the desired action to be performed on a resource:

GET Safe

Retrieves data from the server. Should not modify server state.

GET /api/users/123
POST Unsafe

Sends data to create or process resources on the server.

POST /api/users
PUT Unsafe

Updates or creates a resource at a specific location.

PUT /api/users/123
DELETE Unsafe

Removes the specified resource from the server.

DELETE /api/users/123
PATCH Unsafe

Partially updates a resource with provided data.

PATCH /api/users/123
HEAD Safe

Like GET but returns only headers, no body content.

HEAD /api/users/123

HTTPS - HTTP Secure

HTTPS is the secure version of HTTP, using TLS/SSL encryption to protect data in transit between browsers and servers.

HTTP

Unencrypted data transmission
Vulnerable to eavesdropping
Data can be modified in transit
Faster (no encryption overhead)
Uses port 80

HTTPS

Encrypted data transmission
Protection against eavesdropping
Data integrity verification
Server authentication
SEO benefits

SSL/TLS Handshake Process

1

Client Hello

Browser sends supported cipher suites and SSL/TLS version

2

Server Hello

Server responds with chosen cipher and sends certificate

3

Certificate Verification

Browser verifies server certificate with Certificate Authority

4

Key Exchange

Both parties establish shared encryption keys

5

Secure Communication

Encrypted data transmission begins
