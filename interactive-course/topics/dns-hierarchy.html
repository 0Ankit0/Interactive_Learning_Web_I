DNS and Its Hierarchy

Discover how domain names are translated into IP addresses across the internet

What is DNS?

DNS (Domain Name System) is the phonebook of the Internet. It translates human-readable domain names like "example.com" into machine-readable IP addresses like "192.168.1.1" that computers use to communicate.

Without DNS

Users would need to remember:

142.250.184.14 for Google 151.101.1.140 for Reddit 185.199.108.153 for GitHub

With DNS

Users can simply type:

google.com reddit.com github.com

DNS Hierarchy Structure

DNS follows a hierarchical, tree-like structure starting from the root and branching down to individual hostnames.

Root (.)
.com

Root Level

The top of the DNS hierarchy, managed by IANA. Represented by a single dot (.) but usually omitted.

Top-Level Domains (TLD)

Generic TLDs (.com, .org, .net) and Country Code TLDs (.us, .uk, .de). Managed by registry operators.

Second-Level Domains (SLD)

The main domain name purchased by organizations or individuals (e.g., "google" in google.com).

Subdomains

Additional levels created by domain owners (e.g., "mail" in mail.google.com).

DNS Resolution Process

When you type a domain name, multiple servers work together to resolve it to an IP address:

1

Browser Cache Check

Browser checks its cache for the domain's IP address

If found, resolution complete!
2

OS Cache Check

Operating system checks its DNS cache

Includes hosts file lookup
3

Recursive Resolver

ISP's DNS server receives the query

Acts as intermediary for the client
4

Root Nameserver

Query sent to root nameserver

Returns TLD nameserver address
5

TLD Nameserver

Query sent to TLD nameserver (.com, .org, etc.)

Returns authoritative nameserver
6

Authoritative Nameserver

Final query to authoritative server

Returns the actual IP address

DNS Record Types

Different types of DNS records serve various purposes in the domain name system:

A

Address Record

Maps a domain name to an IPv4 address

example.com. IN A 192.0.2.1
AAAA

IPv6 Address Record

Maps a domain name to an IPv6 address

example.com. IN AAAA 2001:db8::1
CNAME

Canonical Name

Creates an alias pointing to another domain name

www.example.com. IN CNAME example.com.
MX

Mail Exchange

Specifies mail server for email delivery

example.com. IN MX 10 mail.example.com.
NS

Name Server

Delegates a subdomain to a set of name servers

example.com. IN NS ns1.example.com.
TXT

Text Record

Contains arbitrary text, often for verification

example.com. IN TXT "v=spf1 include:_spf.google.com ~all"
