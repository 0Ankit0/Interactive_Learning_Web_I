WCAG 2.1 Guidelines

The Web Content Accessibility Guidelines (WCAG) 2.1 provides a comprehensive framework for making web content accessible to people with disabilities. These guidelines are built on four main principles.

Perceivable

Information must be presentable to users in ways they can perceive.

  • Text Alternatives: Provide text alternatives for non-text content
  • Captions: Add captions to videos
  • Color Contrast: Ensure sufficient contrast ratios
  • Adaptable Content: Use semantic HTML

Operable

Interface components must be operable by all users.

  • Keyboard Access: All functions via keyboard
  • No Seizures: Avoid flashing content
  • Time Limits: Provide time controls
  • Navigation: Clear navigation structure

Understandable

Information and UI operation must be understandable.

  • Readable Text: Use clear language
  • Predictable: Consistent navigation
  • Input Assistance: Help with form errors
  • Language: Identify page language

Robust

Content must work with various assistive technologies.

  • Valid HTML: Use proper markup
  • Compatible: Work with screen readers
  • Future-proof: Technology-agnostic
  • Standards: Follow web standards

Semantic HTML for Accessibility

Semantic HTML provides meaningful structure to web content, making it easier for assistive technologies to understand and navigate your pages.

What is Semantic HTML?

Semantic HTML uses tags that convey meaning about the content they contain, rather than just how it should look. This helps screen readers and other assistive technologies understand the structure and purpose of your content.

Good Practice - Semantic HTML

<header>
  <nav>
    <ul>
      <li><a href="#home">Home</a></li>
      <li><a href="#about">About</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h1>Page Title</h1>
    <p>Content here...</p>
  </article>
</main>

<footer>
  <p>© 2025 Company Name</p>
</footer>

Poor Practice - Non-Semantic HTML

<div class="header">
  <div class="nav">
    <div class="list">
      <div><a href="#home">Home</a></div>
      <div><a href="#about">About</a></div>
    </div>
  </div>
</div>

Color Contrast Requirements

Proper color contrast ensures text is readable for users with visual impairments or color blindness. WCAG 2.1 defines minimum contrast ratios for different text sizes.

Contrast Ratio Requirements

Level AA (Minimum)

Normal text: 4.5:1 contrast ratio

Large text: 3:1 contrast ratio

Required for: Most websites and applications

Level AAA (Enhanced)

Normal text: 7:1 contrast ratio

Large text: 4.5:1 contrast ratio

Recommended for: Enhanced accessibility

ARIA Attributes in Action

ARIA (Accessible Rich Internet Applications) attributes enhance the accessibility of dynamic web content and custom user interface components.

What is ARIA?

ARIA is a set of attributes that define ways to make web content and applications more accessible to people with disabilities. It especially helps with dynamic content and advanced user interface controls developed with JavaScript.

ARIA Roles

Define what an element is or does

<div role="navigation">
  <!-- nav content -->
</div>

<button role="tab">Tab 1</button>

ARIA States

Describe the current state of elements

<button 
  aria-pressed="true"
  aria-expanded="false">
  Toggle
</button>

ARIA Properties

Provide additional information about elements

<input 
  aria-label="Search"
  aria-required="true"
  aria-describedby="help-text"
/>
