Understanding JavaScript Errors

Error handling is a crucial aspect of JavaScript programming that allows you to anticipate, catch, and manage runtime errors gracefully. Proper error handling prevents applications from crashing and provides users with meaningful feedback.

Syntax Errors

Errors in code structure that prevent parsing and execution. 

// Syntax Error - Missing closing parenthesis
console.log("Hello World";

Runtime Errors

Errors that occur during code execution, such as accessing undefined properties. 

// Runtime Error - Cannot read property of undefined
let obj;
console.log(obj.name); // TypeError

Logical Errors

Code runs without throwing errors but produces incorrect results. 

// Logical Error - Wrong operator
function add(a, b) {
  return a - b; // Should be a + b
}

Network Errors

Errors related to network requests, API calls, or connectivity issues. 

// Network Error - Failed fetch request
fetch('/api/data')
  .catch(error => console.error('Network error:', error));

Try-Catch-Finally

The try-catch-finally statement is the primary mechanism for handling errors in JavaScript.

Basic Syntax

try {
  // Code that might throw an error
  let result = riskyOperation();
  console.log(result);
} catch (error) {
  // Handle the error
  console.error('An error occurred:', error.message);
} finally {
  // Always executed (optional)
  console.log('Cleanup operations');
}

Error Objects and Types

JavaScript provides several built-in error types, each designed for specific error scenarios.

Error (Base Class)

Generic error object, parent of all other error types. 

throw new Error('Something went wrong');
SyntaxError

Thrown when there's a syntax error in the code.

// This will throw a SyntaxError
eval('const x = ;');
ReferenceError

Thrown when trying to access an undefined variable.

// This will throw a ReferenceError
console.log(undefinedVariable);
TypeError

Thrown when a value is not of the expected type.

// This will throw a TypeError
let x = null;
x.someMethod();
RangeError

Thrown when a number is outside its allowed range.

// This will throw a RangeError
let arr = new Array(-1);

Creating Custom Error Classes

class ValidationError extends Error {
  constructor(message) {
    super(message);
    this.name = 'ValidationError';
  }
}

class NetworkError extends Error {
  constructor(message, statusCode) {
    super(message);
    this.name = 'NetworkError';
    this.statusCode = statusCode;
  }
}

// Usage
throw new ValidationError('Invalid email format');
throw new NetworkError('Server unavailable', 503);

Async Error Handling

Handling errors in asynchronous code requires special attention, especially with Promises and async/await.

Promise Error Handling

// Using .catch() with Promises
fetch('/api/data')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => {
    console.error('Error fetching data:', error);
    // Handle error (show user message, retry, etc.)
  });

// Chaining multiple catches for different error types
fetch('/api/data')
  .then(response => {
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}: ${response.statusText}`);
    }
    return response.json();
  })
  .catch(error => {
    if (error instanceof TypeError) {
      console.error('Network error:', error);
    } else {
      console.error('HTTP error:', error);
    }
  });

Async/Await Error Handling

async function fetchUserData(userId) {
  try {
    const response = await fetch(`/api/users/${userId}`);
    
    if (!response.ok) {
      throw new Error(`Failed to fetch user: ${response.status}`);
    }
    
    const userData = await response.json();
    return userData;
    
  } catch (error) {
    if (error.name === 'TypeError') {
      // Network error
      console.error('Network connection failed:', error);
      throw new Error('Unable to connect to server');
    } else {
      // Other errors
      console.error('Error fetching user data:', error);
      throw error;
    }
  }
}

// Usage with error handling
try {
  const user = await fetchUserData(123);
  console.log('User data:', user);
} catch (error) {
  displayErrorMessage(error.message);
}

Error Handling Best Practices

Be Specific with Errors

Create meaningful error messages that help identify the problem. 

// Good
throw new Error('User email is required and must be valid');

// Bad
throw new Error('Invalid input');

Use Error Boundaries

Implement centralized error handling to catch unhandled errors. 

window.addEventListener('error', (event) => {
  console.error('Global error:', event.error);
  // Send to logging service
  logError(event.error);
});

window.addEventListener('unhandledrejection', (event) => {
  console.error('Unhandled promise rejection:', event.reason);
  event.preventDefault();
});

Implement Retry Logic

For network errors, implement retry mechanisms with exponential backoff. 

async function fetchWithRetry(url, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fetch(url);
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      await new Promise(resolve => 
        setTimeout(resolve, 1000 * Math.pow(2, i))
      );
    }
  }
}

User-Friendly Messages

Show users helpful error messages, not technical details.

function handleError(error) {
  // Log technical details for developers
  console.error('Technical error:', error);
  
  // Show user-friendly message
  const userMessage = getUserFriendlyMessage(error);
  showNotification(userMessage, 'error');
}

function getUserFriendlyMessage(error) {
  if (error.name === 'NetworkError') {
    return 'Unable to connect. Please check your internet connection.';
  }
  return 'Something went wrong. Please try again later.';
}
