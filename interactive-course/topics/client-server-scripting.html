Understanding the Difference

Web applications use both client-side and server-side scripting to create dynamic, interactive experiences. Understanding where code executes and why is crucial for web development.

Client-Side Scripting

Runs in: Browser Visible to: Everyone
Fast, immediate response
Reduces server load
Rich user interactions
Code is visible/editable
Browser dependent

Server-Side Scripting

Runs on: Web Server Visible to: Server only
Secure code execution
Database access
Platform independent
Requires server resources
Network dependency

Client-Side Technologies

Client-side scripts run in the user's browser and handle user interface interactions, validations, and dynamic content updates.

JavaScript

The primary client-side scripting language for web browsers.

DOM Manipulation Event Handling AJAX Requests Form Validation

Example: Form Validation

function validateEmail(email) {
  const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return regex.test(email);
}

// Real-time validation
document.getElementById('email').addEventListener('input', function() {
  const email = this.value;
  const isValid = validateEmail(email);
  
  this.style.borderColor = isValid ? 'green' : 'red';
  document.getElementById('error').textContent = 
    isValid ? '' : 'Please enter a valid email';
});

Server-Side Technologies

Server-side scripts handle business logic, database operations, user authentication, and generate dynamic content before sending it to the client.

Node.js

JavaScript runtime for server-side development.

Express.js NPM Ecosystem Async/Await API Development

Example: Express Server

const express = require('express');
const app = express();

app.get('/api/users/:id', async (req, res) => {
  try {
    const userId = req.params.id;
    const user = await getUserFromDatabase(userId);
    
    if (!user) {
      return res.status(404).json({ error: 'User not found' });
    }
    
    res.json(user);
  } catch (error) {
    res.status(500).json({ error: 'Server error' });
  }
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
});

Python

Popular language for web development and data processing.

Django Flask FastAPI Data Science

Example: Flask API

from flask import Flask, jsonify, request
import sqlite3

app = Flask(__name__)

@app.route('/api/users', methods=['POST'])
def create_user():
    data = request.get_json()
    
    # Validate input
    if not data.get('email') or not data.get('name'):
        return jsonify({'error': 'Email and name required'}), 400
    
    # Save to database
    conn = sqlite3.connect('users.db')
    cursor = conn.cursor()
    cursor.execute(
        'INSERT INTO users (name, email) VALUES (?, ?)',
        (data['name'], data['email'])
    )
    conn.commit()
    user_id = cursor.lastrowid
    conn.close()
    
    return jsonify({'id': user_id, 'message': 'User created'}), 201

Best Practices & Guidelines

Client-Side Best Practices

Always Validate on Server Too

Client-side validation is for UX, not security

Minimize API Calls

Cache data locally when possible

Handle Errors Gracefully

Provide meaningful error messages to users

Progressive Enhancement

Ensure functionality works without JavaScript

Server-Side Best Practices

Never Trust Client Input

Validate and sanitize all incoming data

Secure Database Queries

Use parameterized queries to prevent SQL injection

Implement Authentication

Verify user identity and permissions

Handle Errors Securely

Don't expose sensitive information in error messages
