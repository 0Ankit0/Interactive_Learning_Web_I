SASS/SCSS Deep Dive

SASS (Syntactically Awesome StyleSheets) is the most popular CSS preprocessor, offering two syntaxes: the indented SASS syntax and the CSS-like SCSS syntax.

Variables

Store values in variables for reuse and easy maintenance.

SASS Variables

SCSS 
// Color variables
$primary: #3498db;
$secondary-color: #2ecc71;
$accent-color: #e74c3c;
$text-color: #2c3e50;

// Size variables
$base-font-size: 16px;
$heading-font-size: $base-font-size * 1.5;
$border-radius: 4px;
$container-width: 1200px;

// Font variables
$font-stack: 'Helvetica Neue', Helvetica, Arial, sans-serif;
$heading-font: 'Georgia', serif;

// Usage
.button {
  background-color: $primary;
  color: white;
  font-family: $font-stack;
  font-size: $base-font-size;
  border-radius: $border-radius;
  padding: 10px 20px;
  
  &:hover {
    background-color: darken($primary, 10%);
  }
}

Nesting

Nest CSS selectors to match your HTML hierarchy for better organization.

Nesting Example

SCSS 
// SCSS with nesting
.navigation {
  background: $primary;
  padding: 1rem;
  
  ul {
    list-style: none;
    margin: 0;
    padding: 0;
    
    li {
      display: inline-block;
      margin-right: 1rem;
      
      a {
        color: white;
        text-decoration: none;
        padding: 0.5rem 1rem;
        border-radius: $border-radius;
        transition: all 0.3s ease;
        
        &:hover {
          background: rgba(white, 0.1);
        }
        
        &.active {
          background: $secondary-color;
        }
      }
    }
  }
  
  // Parent selector reference
  &.fixed {
    position: fixed;
    top: 0;
    left: 0;
    right: 0;
    z-index: 1000;
  }
}

Mixins

Create reusable groups of CSS declarations with optional parameters.

SASS Mixins

SCSS 
// Button mixin with parameters
@mixin button-style($bg-color: $primary, $text-color: white, $size: medium) {
  background: $bg-color;
  color: $text-color;
  border: none;
  border-radius: $border-radius;
  cursor: pointer;
  font-family: $font-stack;
  transition: all 0.3s ease;
  
  @if $size == small {
    padding: 0.5rem 1rem;
    font-size: 0.875rem;
  } @else if $size == medium {
    padding: 0.75rem 1.5rem;
    font-size: 1rem;
  } @else if $size == large {
    padding: 1rem 2rem;
    font-size: 1.125rem;
  }
  
  &:hover {
    background: darken($bg-color, 10%);
    transform: translateY(-2px);
  }
  
  &:active {
    transform: translateY(0);
  }
}

// Flexbox centering mixin
@mixin flex-center($direction: row) {
  display: flex;
  flex-direction: $direction;
  justify-content: center;
  align-items: center;
}

// Media query mixin
@mixin respond-to($breakpoint) {
  @if $breakpoint == tablet {
    @media (max-width: 768px) { @content; }
  }
  @if $breakpoint == mobile {
    @media (max-width: 480px) { @content; }
  }
}

// Usage examples
.primary-btn {
  @include button-style($primary, white, medium);
}

.secondary-btn {
  @include button-style($secondary-color, white, small);
}

.hero-section {
  @include flex-center(column);
  height: 100vh;
  
  @include respond-to(tablet) {
    height: auto;
    padding: 2rem;
  }
}

Functions & Extensions

Use built-in functions and create custom functions for complex calculations.

SASS Functions

SCSS 
// Custom functions
@function rem($pixels) {
  @return #{$pixels / 16}rem;
}

@function color-contrast($color) {
  @if (lightness($color) > 50%) {
    @return #000; // dark text for light backgrounds
  } @else {
    @return #fff; // light text for dark backgrounds
  }
}

// Placeholder selectors (%)
%button-base {
  display: inline-block;
  padding: 0.75rem 1.5rem;
  border: none;
  border-radius: $border-radius;
  cursor: pointer;
  text-decoration: none;
  text-align: center;
  transition: all 0.3s ease;
}

%clearfix {
  &::after {
    content: "";
    display: table;
    clear: both;
  }
}

// Using @extend
.btn-primary {
  @extend %button-base;
  background: $primary;
  color: color-contrast($primary);
  
  &:hover {
    background: darken($primary, 10%);
  }
}

.btn-secondary {
  @extend %button-base;
  background: transparent;
  color: $primary;
  border: 2px solid $primary;
  
  &:hover {
    background: $primary;
    color: color-contrast($primary);
  }
}

// Using custom functions
.container {
  max-width: rem(1200);
  font-size: rem(16);
  line-height: rem(24);
}

// Built-in color functions
.color-variations {
  .original { background: $primary; }
  .lighter { background: lighten($primary, 20%); }
  .darker { background: darken($primary, 20%); }
  .saturated { background: saturate($primary, 30%); }
  .desaturated { background: desaturate($primary, 30%); }
  .complement { background: complement($primary); }
}

LESS Essentials

LESS (Leaner Style Sheets) is a JavaScript-based CSS preprocessor that extends CSS with dynamic behavior such as variables, mixins, operations and functions.

Variables

Use @ symbol for variables, making them easily identifiable and reusable.

Mixins

Create reusable mixins with parameters and guards for conditional logic.

Nesting

Nest rules within other rules to create hierarchical CSS structures.

Operations

Perform mathematical operations and string manipulations in your styles.

LESS Syntax Examples

LESS 
// Variables in LESS (using @)
@primary-color: #3498db;
@secondary-color: #2ecc71;
@base-font-size: 16px;
@border-radius: 4px;

// Mixins with parameters
.border-radius(@radius: @border-radius) {
  border-radius: @radius;
  -webkit-border-radius: @radius;
  -moz-border-radius: @radius;
}

.button-style(@bg: @primary-color, @color: white) {
  background: @bg;
  color: @color;
  .border-radius();
  padding: 10px 20px;
  border: none;
  cursor: pointer;
  
  &:hover {
    background: darken(@bg, 10%);
  }
}

// Nesting and operations
.navigation {
  background: @primary-color;
  padding: @base-font-size / 2;
  
  ul {
    list-style: none;
    margin: 0;
    
    li {
      display: inline-block;
      
      a {
        color: white;
        text-decoration: none;
        padding: (@base-font-size / 4) (@base-font-size / 2);
        
        &:hover {
          background: fade(white, 20%);
        }
      }
    }
  }
}

// Guard expressions (conditionals)
.mixin(@a) when (@a > 10) {
  width: @a * 2px;
}

.mixin(@a) when (@a <= 10) {
  width: @a px;
}

// Usage
.primary-button {
  .button-style(@primary-color, white);
}

.secondary-button {
  .button-style(@secondary-color, white);
}

Build Tools & Workflow

Set up efficient development workflows with popular build tools and task runners for automatic compilation and optimization.

Node.js Setup

1. Install Node.js and npm
# Install SASS globally
npm install -g sass

# Or install locally in your project
npm install --save-dev sass
2. Create package.json
{
  "name": "my-project",
  "version": "1.0.0",
  "scripts": {
    "build-css": "sass src/scss:dist/css",
    "watch-css": "sass --watch src/scss:dist/css",
    "build-css-compressed": "sass --style compressed src/scss:dist/css"
  },
  "devDependencies": {
    "sass": "^1.50.0"
  }
}
3. Run build commands
# Build CSS once
npm run build-css

# Watch for changes
npm run watch-css

# Build compressed CSS for production
npm run build-css-compressed

Webpack Configuration

// webpack.config.js
const path = require('path');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');

module.exports = {
  entry: './src/index.js',
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'bundle.js',
  },
  module: {
    rules: [
      {
        test: /\.scss$/,
        use: [
          MiniCssExtractPlugin.loader,
          'css-loader',
          'sass-loader',
        ],
      },
    ],
  },
  plugins: [
    new MiniCssExtractPlugin({
      filename: 'styles.css',
    }),
  ],
};

Gulp Task Runner

// gulpfile.js
const gulp = require('gulp');
const sass = require('gulp-sass')(require('sass'));
const autoprefixer = require('gulp-autoprefixer');
const cleanCSS = require('gulp-clean-css');
const sourcemaps = require('gulp-sourcemaps');

function styles() {
  return gulp.src('./src/scss/**/*.scss')
    .pipe(sourcemaps.init())
    .pipe(sass().on('error', sass.logError))
    .pipe(autoprefixer())
    .pipe(cleanCSS())
    .pipe(sourcemaps.write('.'))
    .pipe(gulp.dest('./dist/css'));
}

function watch() {
  gulp.watch('./src/scss/**/*.scss', styles);
}

exports.styles = styles;
exports.watch = watch;
exports.default = gulp.series(styles, watch);

Naming Conventions

BEM Methodology

// Block
.card { }

// Block__Element
.card__title { }
.card__content { }

// Block--Modifier
.card--featured { }
.card--large { }

// Block__Element--Modifier
.card__title--highlighted { }

Variable Naming

// Colors
$color-primary: #3498db;
$color-secondary: #2ecc71;
$color-text-light: #666;

// Sizes
$size-xs: 0.5rem;
$size-sm: 0.75rem;
$size-md: 1rem;
$size-lg: 1.25rem;

// Breakpoints
$breakpoint-mobile: 480px;
$breakpoint-tablet: 768px;
$breakpoint-desktop: 1024px;

Performance Optimization

Minimize Nesting

Avoid nesting more than 3-4 levels deep to prevent overly specific selectors and large CSS output.

Use Mixins Wisely

Only use mixins for code that accepts parameters. Use @extend for static code blocks.

Modular Imports

Use partials and @import to keep files manageable and enable team collaboration.

Compile for Production

Always use compressed output and remove source maps for production deployments.
