HTML Tables

Tables for Data Organization

HTML tables are used to display data in rows and columns, making information easy to read and compare. They're perfect for displaying structured data like schedules, statistics, or product comparisons.

Basic Table Structure

<table>
    <thead>
        <tr>
            <th>Name</th>
            <th>Age</th>
            <th>City</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>John Doe</td>
            <td>25</td>
            <td>New York</td>
        </tr>
        <tr>
            <td>Jane Smith</td>
            <td>30</td>
            <td>Los Angeles</td>
        </tr>
    </tbody>
</table>

<table>

The main container element that wraps all table content.

<thead>

Groups header content, typically contains column headings.

<tbody>

Groups the main body content of the table.

<tfoot>

Groups footer content, like totals or summary information.

<tr>

Defines a table row that contains cells.

<th>

Header cell that represents column or row headings.

<td>

Data cell that contains the actual table data.

<caption>

Provides a title or description for the table.

Advanced Table Features

Spanning Rows and Columns

<table>
    <tr>
        <th rowspan="2">Name</th>
        <th colspan="2">Contact</th>
    </tr>
    <tr>
        <th>Email</th>
        <th>Phone</th>
    </tr>
    <tr>
        <td>John Doe</td>
        <td>john@email.com</td>
        <td>123-456-7890</td>
    </tr>
</table>

               HTML Forms
            


            

              
Collecting User Input

              

                HTML forms are used to collect user input and send it to a
                server for processing. They're essential for user registration,
                contact forms, surveys, and any interaction that requires user
                data.
              

            


            

              
Basic Form Structure

              

                
<form action="/submit" method="post">
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" required>

    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>

    <label for="message">Message:</label>
    <textarea id="message" name="message" rows="4"></textarea>

    <button type="submit">Submit</button>
</form>

              

            


            

              

                
action

                
Specifies where to send the form data when submitted

              

              

                
method

                
Defines how to send the data (GET or POST)

              

              

                
enctype

                
Specifies how form data should be encoded

              

              

                
target

                
Determines where to display the response

               Form Validation
            


            

              
Client-Side Validation

              

                HTML5 provides built-in validation attributes that help ensure
                users enter valid data before submitting forms.
              

            


            

              

                
Required Fields

                

                  
<input type="text" name="username" required>

                

                
              

              

                
Pattern Validation

                

                  
<input type="text" pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}" placeholder="123-456-7890">

                

                
              

              

                
Min/Max Length

                

                  
<input type="password" minlength="8" maxlength="20">

                

                
              

              

                
Min/Max Values

                

                  
<input type="number" min="1" max="100">
