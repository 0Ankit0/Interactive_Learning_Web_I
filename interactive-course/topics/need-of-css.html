Introduction to CSS

Learning Objectives

  • Understand the limitations of HTML for styling
  • Recognize the need for separation of content and presentation
  • Identify the benefits CSS brings to web development
  • Appreciate the importance of maintainable code

CSS (Cascading Style Sheets) is a stylesheet language used to describe the presentation of a document written in HTML. While HTML provides the structure and content of a webpage, CSS handles the visual presentation - colors, fonts, layouts, animations, and more.

In the early days of the web, HTML was used for both structure and presentation. However, as websites became more complex, this approach revealed significant limitations that led to the development and adoption of CSS.

Key Insight

CSS follows the principle of separation of concerns - separating content (HTML) from presentation (CSS) for better code organization and maintainability.

Problems Without CSS

1. Inline Styling Limitations

Before CSS, styling was done using HTML attributes and elements like <font>, <center>, and inline styles: 

<!-- Old way: Mixing content with presentation -->
<p><font color="red" size="4" face="Arial">This is a red paragraph</font></p>
<table bgcolor="#cccccc" width="100%" cellpadding="10">
  <tr>
    <td align="center"><font color="blue" size="6">Centered Blue Text</font></td>
  </tr>
</table>

2. Code Repetition

Without CSS, styling information had to be repeated for every element:

Problems with this approach:

  • Repetitive code: Same styling attributes repeated multiple times
  • Large file sizes: More HTML markup means bigger files
  • Difficult maintenance: Changing colors or fonts requires editing every element
  • Inconsistency: Easy to miss elements when making changes
  • Limited functionality: HTML attributes can't create complex layouts or animations

3. Maintenance Nightmare

Imagine having to change the color scheme of a website with hundreds of pages - you'd need to edit every single HTML file and every single styling attribute!

Benefits of CSS

Separation of Concerns

HTML handles structure and content, CSS handles presentation. This makes code more organized and easier to understand.

Reusability

Write CSS once, apply it to multiple elements across multiple pages. Change one rule to update the entire site.

Performance

CSS files can be cached by browsers, reducing load times. Smaller HTML files load faster.

Maintainability

Easy to update styles across entire websites. Consistent styling is easier to achieve and maintain.

Advanced Features

CSS provides powerful features like animations, transitions, responsive design, and complex layouts.

Accessibility

Better semantic HTML with CSS styling improves accessibility for screen readers and other assistive technologies.

Important Notice
Problems with this approach:
  • Repetitive font tags for every element
  • Hard to maintain - need to change every element individually
  • Mixing presentation with content
  • Larger file size due to repeated styling code
  • Inconsistent styling is common

Modern Way: CSS Styling

HTML Code:
<h1 class="main-heading">Welcome to My Site</h1>
<p class="content-text">This is a paragraph with CSS styling.</p>
<p class="content-text">Another paragraph with the same styling.</p>
<div class="notice-box">
  <p class="notice-text">Important Notice</p>
</div>
CSS Code:
.main-heading {
  color: #2c3e50;
  font-size: 2.5rem;
  font-family: Arial, sans-serif;
  margin-bottom: 1rem;
}

.content-text {
  color: #34495e;
  font-size: 1rem;
  font-family: Arial, sans-serif;
  line-height: 1.6;
  margin-bottom: 1rem;
}

.notice-box {
  background-color: #ecf0f1;
  padding: 15px;
  border-radius: 5px;
  margin-top: 20px;
}

.notice-text {
  color: #e74c3c;
  font-size: 1.2rem;
  font-family: Arial, sans-serif;
  font-weight: bold;
  margin: 0;
}
Result:

Welcome to My Site

This is a paragraph with CSS styling.

Another paragraph with the same styling.

Important Notice

Benefits of this approach:
  • Clean, semantic HTML that describes content structure
  • Reusable CSS classes that can be applied to multiple elements
  • Easy to maintain - change CSS once, update entire site
  • Better performance - CSS can be cached by browsers
  • More powerful styling options (gradients, animations, responsive design)

Scenario: Changing the Primary Color

Without CSS (Old Way)

To change from blue to green:

  • Edit 47 HTML files
  • Change 156 individual color attributes
  • Risk missing some elements
  • Time required: 2-3 hours
  • Risk of inconsistencies: High
With CSS (Modern Way)

To change from blue to green:

  • Edit 1 CSS file
  • Change 1 CSS variable or property
  • All elements update automatically
  • Time required: 30 seconds
  • Risk of inconsistencies: None
