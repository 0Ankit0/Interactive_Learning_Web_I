What is Semantic HTML?

Semantic vs Non-Semantic

Semantic HTML uses elements that clearly describe their meaning to both browsers and developers. Instead of generic <div> elements, semantic elements like <header>, <article>, and <footer> tell us exactly what content they contain.

Why Semantic HTML Matters:

  • Accessibility: Screen readers understand the content structure
  • SEO: Search engines better understand your content
  • Maintainability: Code is easier to read and modify
  • Styling: CSS can target semantic elements directly
  • Future-proof: Works better with new technologies

Structural Elements

Main Structural Elements

<header>

Contains introductory content, site branding, navigation

Usage: Page header, article header, section header

<nav>

Contains navigation links for the site or page section

Usage: Main navigation, breadcrumbs, page links

<main>

Contains the main content of the document

Usage: Primary content area (only one per page)

<section>

Groups related content together with a heading

Usage: Chapters, tabs, themed content groups

<article>

Self-contained content that could stand alone

Usage: Blog posts, news articles, comments

<aside>

Content indirectly related to main content

Usage: Sidebars, callout boxes, related links

<footer>

Contains footer information for its nearest ancestor

Usage: Page footer, article footer, copyright info

SEO Advantages

Search Engine Optimization

Search engines use semantic HTML to better understand your content structure and context, which can improve your search rankings and visibility.

How Semantic HTML Improves SEO

Better Crawling

Search engines can more easily understand and index your content structure.

Rich Snippets

Semantic markup enables rich snippets in search results, improving click-through rates.

Mobile-First

Semantic HTML works better with mobile-first indexing and responsive design.

Social Sharing

Proper markup improves how your content appears when shared on social media.
