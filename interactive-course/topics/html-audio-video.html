Advanced Controls

Custom controls, preloading strategies, and programmatic audio manipulation.

Basic Audio Element

HTML 
<!-- Simple audio with controls -->
<audio controls>
    <source src="audio/sample.mp3" type="audio/mpeg">
    <source src="audio/sample.ogg" type="audio/ogg">
    <p>Your browser doesn't support HTML5 audio. 
       <a href="audio/sample.mp3">Download the audio</a> instead.</p>
</audio>

<!-- Audio with additional attributes -->
<audio controls preload="metadata" loop>
    <source src="audio/background.mp3" type="audio/mpeg">
    <source src="audio/background.wav" type="audio/wav">
    Your browser does not support the audio element.
</audio>

Audio Element Attributes

Attribute Description Example
controls Shows default audio controls <audio controls>
autoplay Starts playing automatically (use cautiously) <audio autoplay>
loop Repeats audio when finished <audio loop>
muted Starts with audio muted <audio muted>
preload How much to preload (none, metadata, auto) <audio preload="metadata">
src Direct source URL <audio src="audio.mp3">

HTML Video Element

The HTML <video> element enables native video playback in browsers with comprehensive controls, accessibility features, and responsive capabilities.

Native Playback

Built-in browser video playback without plugins or external dependencies.

Responsive Design

Automatic scaling and responsive behavior for different screen sizes.

Accessibility

Built-in support for captions, subtitles, and screen reader compatibility.

Basic Video Element

HTML 
<!-- Basic video with multiple sources -->
<video controls width="600" height="400">
    <source src="video/sample.mp4" type="video/mp4">
    <source src="video/sample.webm" type="video/webm">
    <source src="video/sample.ogv" type="video/ogg">
    <p>Your browser doesn't support HTML5 video. 
       <a href="video/sample.mp4">Download the video</a> instead.</p>
</video>

<!-- Video with poster and tracks -->
<video controls poster="images/video-poster.jpg" preload="metadata">
    <source src="video/lecture.mp4" type="video/mp4">
    <source src="video/lecture.webm" type="video/webm">
    
    <!-- Subtitle tracks -->
    <track kind="subtitles" src="subtitles/english.vtt" srclang="en" label="English">
    <track kind="subtitles" src="subtitles/spanish.vtt" srclang="es" label="Español">
    
    <!-- Fallback content -->
    <p>Your browser doesn't support HTML5 video.</p>
</video>

Format Support and Best Practices

Understanding format compatibility and optimization strategies for maximum reach and performance.

Audio Formats

MP3

Excellent

Compatibility: Universal browser support

Quality: Good compression, acceptable quality

Use Case: Primary format for web audio

OGG

Good

Compatibility: Firefox, Chrome, Opera

Quality: Better compression than MP3

Use Case: Fallback for open source browsers

WAV

Limited

Compatibility: Most browsers

Quality: Uncompressed, large files

Use Case: High-quality audio, short clips

Video Formats

MP4 (H.264)

Excellent

Compatibility: Universal browser support

Quality: Excellent compression and quality

Use Case: Primary format for web video

WebM

Good

Compatibility: Chrome, Firefox, Opera

Quality: Good compression, open standard

Use Case: Fallback for open source browsers

OGV

Limited

Compatibility: Firefox, older browsers

Quality: Decent compression

Use Case: Legacy browser support

Best Practices

Mobile Optimization

  • Use playsinline attribute for iOS
  • Provide multiple quality options
  • Consider data usage implications
  • Test autoplay policies on mobile

Accessibility

  • Always provide captions/subtitles
  • Include transcript alternatives
  • Use descriptive poster images
  • Ensure keyboard navigation works

Performance

  • Use preload="metadata" by default
  • Optimize file sizes and compression
  • Consider lazy loading for below-fold content
  • Implement progressive enhancement

User Experience

  • Respect user's autoplay preferences
  • Provide clear play/pause controls
  • Show loading states and buffering
  • Handle errors gracefully

Advanced Multimedia Features

JavaScript Media API

JavaScript 
// Get media element
const video = document.getElementById('myVideo');

// Event listeners
video.addEventListener('loadstart', () => {
    console.log('Started loading');
});

video.addEventListener('canplay', () => {
    console.log('Can start playing');
});

video.addEventListener('timeupdate', () => {
    const progress = (video.currentTime / video.duration) * 100;
    updateProgressBar(progress);
});

// Control functions
function playVideo() {
    video.play().catch(error => {
        console.error('Error playing video:', error);
    });
}

function pauseVideo() {
    video.pause();
}

function seekTo(seconds) {
    video.currentTime = seconds;
}

function setVolume(level) {
    video.volume = Math.max(0, Math.min(1, level));
}

// Advanced features
function requestFullscreen() {
    if (video.requestFullscreen) {
        video.requestFullscreen();
    } else if (video.webkitRequestFullscreen) {
        video.webkitRequestFullscreen();
    }
}

function captureFrame() {
    const canvas = document.createElement('canvas');
    const ctx = canvas.getContext('2d');
    canvas.width = video.videoWidth;
    canvas.height = video.videoHeight;
    ctx.drawImage(video, 0, 0);
    return canvas.toDataURL();
}

Responsive Video CSS

CSS 
/* Responsive video container */
.video-container {
    position: relative;
    width: 100%;
    height: 0;
    padding-bottom: 56.25%; /* 16:9 aspect ratio */
    overflow: hidden;
}

.video-container video {
    position: absolute;
    top: 0;
    left: 0;
    width: 100%;
    height: 100%;
    object-fit: cover; /* or 'contain' for letterboxing */
}

/* Mobile-first responsive design */
@media (max-width: 768px) {
    .video-container {
        padding-bottom: 75%; /* 4:3 for mobile */
    }
    
    video {
        max-width: 100%;
        height: auto;
    }
}

/* Custom controls styling */
.custom-controls {
    display: flex;
    align-items: center;

    padding: 10px;

    color: white;
}

.progress-bar {
    flex: 1;
    height: 5px;
    background: rgba(255, 255, 255, 0.3);
    border-radius: 3px;
    overflow: hidden;
}

.progress-bar .progress {
    height: 100%;
    background: #007bff;
    transition: width 0.1s;
}

Exercise 2: Custom Video Controls

Intermediate

Objective: Build custom video controls using JavaScript.

Requirements:

  • Hide default controls and create custom ones
  • Implement play/pause, volume, and progress controls
  • Add fullscreen functionality
  • Display current time and duration

JavaScript Implementation:

const video = document.getElementById('customVideo');
const playBtn = document.getElementById('playBtn');
const progressBar = document.getElementById('progressBar');
const timeDisplay = document.getElementById('timeDisplay');

playBtn.addEventListener('click', () => {
    if (video.paused) {
        video.play();
        playBtn.innerHTML = '⏸️';
    } else {
        video.pause();
        playBtn.innerHTML = '▶️';
    }
});

video.addEventListener('timeupdate', () => {
    const progress = (video.currentTime / video.duration) * 100;
    progressBar.value = progress;
    
    const current = formatTime(video.currentTime);
    const duration = formatTime(video.duration);
    timeDisplay.textContent = `${current} / ${duration}`;
});

function formatTime(seconds) {
    const mins = Math.floor(seconds / 60);
    const secs = Math.floor(seconds % 60);
    return `${mins}:${secs.toString().padStart(2, '0')}`;
}
